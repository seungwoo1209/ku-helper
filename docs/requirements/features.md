## 스펙 변경 이력

- **2026-05-20 — F-13 전체 열람실 합산 옵션 추가**: 원 스펙은 "열람실별" 임계값만 정의했으나, 개별 열람실 외에 **전체 열람실 합산 잔여 좌석**에 대한 임계값 알림을 추가했다. 알림 설정의 `reading_room_id=0` 이 전체 합산을 의미하며(1·2·3·5 는 개별 열람실, 제4열람실 미운영), 임계값 평가·중복 방지(F-14)·긴급 표시(F-15) 동작은 개별 열람실과 동일하다. 이는 이벤트 기반 임계값 알림의 확장이며, Out of Scope 의 "도서관 정기 현황 알림"(주기적 현황 브로드캐스트)과는 별개다.
- **2026-05-23 — F-08·F-10 범위 제외**:
  - **F-08(지하철 혼잡도·지연 정보)**: 서울 공공 API 가 정식 혼잡도(여유/보통/혼잡) 필드를 제공하지 않아 데이터 소스가 부재. `arvlCd` 진입/도착/출발 라벨은 F-07 임베드에 이미 반영되어 있고, 그 이상의 혼잡도·지연 사유 노출은 외부 데이터 소스 확보 시점까지 보류. Out of Scope 로 이전.
  - **F-10(점심 추천 가격 필터)**: 네이버 지역검색 API 가 가격 데이터를 제공하지 않아 필터링 근거 자체가 없음. 가격 정보를 별도 수동 큐레이션할 의향이 없으므로 Out of Scope 로 이전. F-09 임베드에서도 "가격대" 필드를 제거(식당명·대표 메뉴·거리만 노출).
