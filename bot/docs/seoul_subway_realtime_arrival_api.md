# 서울시 지하철 실시간 도착정보 API

> 서울 열린데이터광장 / TOPIS(서울시 교통정보과) 제공
> 서비스명(SERVICE 파라미터 값): `realtimeStationArrival`

## 1. 개요

서울시 교통정보과[TOPIS]에서 제공하는 정보를 이용한 지하철 실시간 도착정보 조회 API.

- 서울시 이외의 역 구간은 미제공 (예: 광명, 서동탄, 춘천 등)
- 데이터 라이선스: 공공누리 1유형 — 출처표시, 상업적 이용 및 변경 가능
- 최종수정일: 2026.05.19 / 생성기준일: 2015.06.17

## 2. Endpoint

```
http://swopenAPI.seoul.go.kr/api/subway/{KEY}/{TYPE}/realtimeStationArrival/{START_INDEX}/{END_INDEX}/{statnNm}
```

샘플 URL (인증키 자리에 `sample` 사용, 최대 5건):

```
http://swopenAPI.seoul.go.kr/api/subway/sample/xml/realtimeStationArrival/0/5/서울
```

## 3. 요청 인자 (경로 파라미터, 순서대로)

| 변수 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `KEY` | STRING | 필수 | OPENAPI에서 발급된 인증키 |
| `TYPE` | STRING | 필수 | 응답 포맷: `xml`, `xmlf`(xml파일), `xls`(엑셀파일), `json` |
| `SERVICE` | STRING | 필수 | 고정값 `realtimeStationArrival` |
| `START_INDEX` | INTEGER | 필수 | 요청 시작 위치 (정수) |
| `END_INDEX` | INTEGER | 필수 | 요청 종료 위치 (정수) |
| `statnNm` | STRING | 필수 | 지하철역명 (예: `서울`, `강남`) |

요청 건수 제한: `END_INDEX - START_INDEX` ≤ 1000. 샘플키(`sample`)는 인덱스 1~5만 허용.

### 역 이름 조회 시 주의 (실제 데이터상 명칭)

| 입력하려는 역 | 실제 명칭 |
|---|---|
| 응암 | 응암순환(상선) |
| 공릉 | 공릉(서울산업대입구) |
| 춘의 | 춘의 |
| 남한산성입구 | 남한산성입구(성남법원, 검찰청) |
| 대모산입구 | 대모산 |
| 천호 | 천호(풍납토성) |
| 몽촌토성 | 몽촌토성(평화의문) |

## 4. 출력값

| 순번 | 필드명 | 설명 |
|---|---|---|
| 1 | `subwayId` | 지하철 호선 ID (아래 코드표 참조) |
| 2 | `updnLine` | 상하행선 구분 (상행/내선, 하행/외선) |
| 3 | `trainLineNm` | 도착지 방면. 예: `성수행(목적지역) - 구로디지털단지방면(다음역)` |
| 5 | `statnFid` | 이전 지하철역 ID |
| 6 | `statnTid` | 다음 지하철역 ID |
| 7 | `statnId` | 지하철역 ID |
| 8 | `statnNm` | 지하철역명 |
| 9 | `trnsitCo` | 환승 노선 수 |
| 10 | `ordkey` | 도착예정 열차순번. 형식: 상하행코드(1) + 순번(1) + 첫 도착예정정류장−현재정류장(3) + 목적지정류장 + 급행여부(1) |
| 11 | `subwayList` | 연계 호선 ID (예: `1002,1007`) |
| 12 | `statnList` | 연계 지하철역 ID (예: `1002000233,1007000000`) |
| 13 | `btrainSttus` | 열차 종류 (급행, ITX, 일반, 특급) |
| 14 | `barvlDt` | 열차 도착예정시간 (단위: 초) |
| 15 | `btrainNo` | 열차번호 (호선별 현재 운행 열차번호) |
| 16 | `bstatnId` | 종착 지하철역 ID |
| 17 | `bstatnNm` | 종착 지하철역명 |
| 18 | `recptnDt` | 열차 도착정보를 생성한 시각 (보정 기준, 아래 주의 참조) |
| 19 | `arvlMsg2` | 첫 번째 도착 메시지 (도착, 출발, 진입 등) |
| 20 | `arvlMsg3` | 두 번째 도착 메시지 (예: `종합운동장 도착`, `12분 후 (광명사거리)`) |
| 21 | `arvlCd` | 도착코드 (아래 코드표 참조) |
| 22 | `lstcarAt` | 막차 여부 (1: 막차, 0: 아님) |

> 순번 4는 원본 명세에 존재하지 않음 (원문 그대로).

## 5. 시차 보정 주의사항 (구현 시 필수)

원천에서 데이터가 수집·가공되어 서비스되는 과정에서 시간 차가 발생할 수 있음.
`recptnDt`(도착정보 생성 시각)와 현재 시각의 차이만큼 열차가 더 진행한 것으로 보정해서 사용해야 함.

> 예: 현재 10:05:30, `recptnDt` 10:03:30 → 2분 시차 → 도착정보를 2분 당기거나 1개 역 더 진행한 것으로 판단.

## 6. 에러 / 정보 메시지

| 코드 | 의미 |
|---|---|
| INFO-000 | 정상 처리 |
| INFO-100 | 인증키가 유효하지 않음 (열린데이터광장에서 신청) |
| INFO-200 | 해당하는 데이터 없음 |
| ERROR-300 | 필수 값 누락 |
| ERROR-301 | 파일타입(TYPE) 값 누락 혹은 유효하지 않음 |
| ERROR-310 | 해당 서비스(SERVICE) 없음 |
| ERROR-331 | START_INDEX 값 확인 필요 |
| ERROR-332 | END_INDEX 값 확인 필요 |
| ERROR-333 | 요청위치 값 타입 오류 (정수 입력) |
| ERROR-334 | START_INDEX가 END_INDEX보다 큼 |
| ERROR-335 | 샘플키는 1회 최대 5건 (인덱스 1~5) |
| ERROR-336 | 1회 최대 1000건 초과 |
| ERROR-500 | 서버 오류 |
| ERROR-600 | DB 연결 오류 |
| ERROR-601 | SQL 문장 오류 |
